Notable Highlights
◆
Roadmap shift: Headless API & GraphQL research injected. As Mighty moves forward with GraphQL and the headless API project, mobile needs to determine the approach and architecture to adopt the new API for each platform — especially iOS and how we move past Core Data. Andrej & Sammy kick off a week-long spike Apr 7; findings review Apr 13, proposal Apr 14. Feed refactor continues in parallel behind feature flag.
◆
Post Detail: rewrite over bridge decision. Rodrigo's analysis confirmed 98% of feed card components are identical to Post Detail — so both platforms will use a single component with a detail view mode, eliminating throwaway bridging work.
◆
Figma → Code pipeline is live. Android can now go from Figma designs straight to production Compose components. iOS deployed matching skills: ios-figma-mapper, ios-ui-component, ios-feed-content. Each new component accelerates faster than the last!
◆
Claude skills compounding. Both platforms have AI skills that encode architecture knowledge (data layer patterns, migration playbooks, DTO scaffolding). New feed screens get ~85-90% of boilerplate generated automatically.
◆
Feed prompt card tracking: engagement dashboard in the works (Colleen working with Kelsey).
◆
iOS icon migration underway. Cleaning out ~200 legacy icons and removing the friction of reviewing multiple icon systems — which introduced bugs, naming variations, and unnecessary decision-making across the app.
◆
Connecting with Mammoth 4/1 on coordination efforts ahead of their editorial feed projects.
